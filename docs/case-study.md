## Case study problem

Cloud and security teams repeatedly face the same operational questions:

- Are Conditional Access, MFA, and risky sign-in controls correctly designed?
- Are users allowed to grant risky OAuth consent to external apps?
- Are mailbox rules, MFA methods, and enterprise app consents checked after suspected compromise?
- Is SharePoint oversharing cleaned before Copilot rollout?
- Are Purview labels, DLP, and information protection questions handled before AI search reaches sensitive data?
- Are cloud cost controls in place for Azure OpenAI / GenAI workloads?
- Are runbooks, notes, decisions, and evidence preserved across days/weeks instead of disappearing into chat history?

Most teams solve these with manual checklists, ad hoc scripts, and scattered notes. AI can help, but only if it is operationalized safely.

## Proposed solution

A private Hermes/OpenClaw-style AI operations assistant running on a Linux VPS.

### Core components

- **Telegram interface:** fast operational intake and notification channel.
- **Hermes-style orchestration:** tasks broken into sub-agents/runbooks rather than one long prompt.
- **Local memory:** markdown/Obsidian-style notes for decisions, reusable procedures, evidence, and context.
- **Security modules:** M365/Azure security readiness, Copilot readiness, identity hardening checklist, FinOps/cost-control prompts.
- **Human approval gates:** no irreversible tenant changes, credential use, outreach, or external actions without approval.
- **Evidence-first reporting:** summaries include artifact locations, blockers, verification status, and next actions.

## Example workflow

### Workflow: Copilot / M365 readiness triage

1. User asks via Telegram: “Prepare a Copilot readiness checklist for a mid-sized M365 tenant.”
2. Agent retrieves the current internal runbook and prior notes.
3. Agent creates structured checklist sections:
   - SharePoint permissions / oversharing
   - Purview labels
   - DLP / Insider Risk / AI data controls
   - Entra ID Conditional Access
   - external app consent governance
   - mailbox compromise checks
   - user communication/training
4. Agent marks any item requiring real tenant access as **manual verification required**.
5. Agent produces a public-safe report and optionally a technical runbook.

### Workflow: Identity hardening mini-assessment

1. Ask for a short assessment template.
2. Agent produces checks for:
   - Conditional Access policy coverage
   - MFA registration state
   - privileged role protection
   - risky sign-in handling
   - user consent restrictions
   - enterprise applications review
   - mailbox rules and compromise indicators
3. Output is formatted as:
   - executive summary
   - risks
   - recommended next actions
   - manual verification list

### Workflow: FinOps / GenAI cost-control reminder

1. Agent checks/update runbook for Azure OpenAI and GenAI cost-control questions.
2. Output highlights:
   - no simple “pause deployment” assumption
   - possible APIM / Logic App / Function / network gate options
   - budget alerts and usage review
   - human approval before deleting or changing resources
