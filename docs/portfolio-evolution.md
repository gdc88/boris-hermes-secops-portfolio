# Portfolio evolution

These repositories form a deliberately evolving portfolio. The goal is to show growth from early AI-assisted career tools into practical, safer AI automation for IT, cloud, and security operations.

## Portfolio map

- **[Hermes SecOps Copilot](https://github.com/gdc88/boris-hermes-secops-portfolio)** — Newest portfolio layer: Hermes/OpenClaw-style AI automation for cloud security operations, M365/Azure readiness, Copilot governance, and agentic workflows.
- **[AI Automation Ops Lab](https://github.com/gdc88/boris-ai-automation-ops-lab)** — Operational base layer: self-hosted AI automation patterns, Telegram delivery, scheduled agents, browser-assisted workflows, and infrastructure operations thinking.
- **[Ops Agent Playbook Runner](https://github.com/gdc88/ops-agent-playbook-runner)** — Engineering proof layer: safe, auditable, dry-run-first operations playbooks with evidence bundles and policy controls.
- **[AI Resume Adapter Bot](https://github.com/gdc88/ai-resume-adapter-bot)** — Career automation layer: ATS/job-description analysis and truthful resume tailoring workflow for the German market.
- **[JobMatch AI](https://github.com/gdc88/JobMatch-AI)** — Course/final-project layer: static MVP for job-match analysis, outreach draft generation, and portfolio demonstration.

## How to keep the portfolio current

- Add short evolution notes when a project gains a new capability, safer workflow, better visual, or clearer technical positioning.
- Keep sensitive data out of GitHub: no tokens, private Telegram/Gmail/Sheets data, browser sessions, tenant exports, or screenshots with personal records.
- Link related projects so the portfolio shows a system of growth, not isolated experiments.
- Prefer proof artifacts: live pages, PDFs, architecture diagrams, tests, CI status, demo walkthroughs, and sanitized screenshots.
